<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| job output → DB | Job writes signal rows and run rows — malformed data would persist silently |
| Finnhub API → headlines | News headlines embedded in Claude prompts — prompt injection risk |
| Claude API response → DB | Parsed classification output persisted to signals table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Information Disclosure | Telegram digest content | accept | Bot delivers to TELEGRAM_CHAT_ID only; operator controls bot permissions; chat is private |
| T-07-08 | Tampering | Prompt injection via Finnhub headlines | mitigate | Existing code strips control characters, caps at ~500 chars/headline, uses &lt;headline&gt; delimiters — confirmed in CLAUDE.md and implemented in v1.0 |
| T-07-09 | Denial of Service | Finnhub rate limit (60 calls/min) | mitigate | Discovery agent uses stdlib token bucket; sequential jobs avoid concurrent-access stacking; if 429 received, tenacity retries with backoff |
| T-07-10 | Repudiation | No evidence of job failure if heartbeat config wrong | mitigate | runs table status=failed is the fallback — operator can audit DB even if healthchecks.io missed the /fail ping |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this ops-only phase |
</threat_model>
