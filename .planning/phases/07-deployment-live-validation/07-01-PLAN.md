<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env → config.py | Secrets loaded from plaintext file on disk |
| Terminal → external APIs | Credentials transmitted over TLS to Finnhub, Telegram, Anthropic, healthchecks.io |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | .env file on disk | mitigate | Confirm `git status .env` shows no staged changes; .gitignore must cover it |
| T-07-02 | Spoofing | Telegram bot token | mitigate | Token is sent over TLS; TELEGRAM_CHAT_ID scopes delivery to correct chat only |
| T-07-03 | Tampering | healthchecks.io UUID | accept | UUID is not secret — compromise only allows false success pings, which is low risk for a personal system |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this ops-only phase |
</threat_model>
