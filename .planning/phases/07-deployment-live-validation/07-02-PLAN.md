    ops/tasks/discovery.xml — Task name: "\Signal System\Discovery", schedule: Mon-Fri 7:30 AM MST, timeout: 45 min
    All three XML files use:
      Command: C:\Users\alexe\.local\bin\uv.exe
      WorkingDirectory: C:\Users\alexe\trade-radar
      LogonType: InteractiveToken (runs as your user session — machine must be logged in)

  <action>
Method A — PowerShell (preferred, scriptable, run as Administrator):

  # Import all three tasks via schtasks
  schtasks /Create /XML "C:\Users\alexe\trade-radar\ops\tasks\daily-close.xml" /TN "\Signal System\Daily Close" /F
  schtasks /Create /XML "C:\Users\alexe\trade-radar\ops\tasks\news-morning.xml" /TN "\Signal System\News Morning" /F
  schtasks /Create /XML "C:\Users\alexe\trade-radar\ops\tasks\discovery.xml" /TN "\Signal System\Discovery" /F

The /F flag overwrites an existing task with the same name (safe to re-run).

Method B — Task Scheduler UI (alternative if schtasks fails):
  1. Open Task Scheduler (taskschd.msc)
  2. Right-click "Task Scheduler Library" → "Import Task..."
  3. Browse to C:\Users\alexe\trade-radar\ops\tasks\daily-close.xml
  4. Review the task details and click OK
  5. Repeat for news-morning.xml and discovery.xml
  6. After import, expand "\Signal System\" in the left pane to confirm all three tasks appear

Note: If importing requires Administrator, right-click Task Scheduler and "Run as administrator".
  </action>
