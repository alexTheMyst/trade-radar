## Overview

Built on the completed MVP Week 1 infrastructure (DB, heartbeat, email, config, dispatcher), this milestone delivers the two AI agents (News Classifier + Discovery Agent), the Alert Router, and the job orchestrators that form the complete signal system. Work proceeded bottom-up by dependency: shared types and foundation first, data-layer extensions second, agents third (News Classifier and Discovery Agent built in parallel), router fourth, and thin job orchestrators last.

The system is implementation-complete. Operators can run `news-morning` and `discovery` jobs that emit heartbeat-monitored, budget-capped alerts to their inbox. Remaining blockers are limited to four manual go-live evidence items in the operator's Windows/Gmail/Healthchecks environment.

- `hashlib.md5` for universe partitioning ‚Äî built-in `hash()` is process-salted

## Milestone Summary

### Key Decisions

| Decision | Rationale |
|----------|-----------|
| Frozen dataclass (not Pydantic) for `Signal` | No untrusted input, simpler, faster; Pydantic reserved for thesis taxonomy where validation matters |
| SHA-256 for `alert_id` (not UUID) | Deterministic across reruns ‚Äî same alert on same day produces same ID, enables `INSERT OR IGNORE` safety |
| `messages.parse()` (not `instructor`) | SDK-native structured output as of current `anthropic` SDK; no extra dependency |
| Sequential execution (no `asyncio`) | Windows event-loop quirks; 500 tickers/day √ó 1-2 calls/ticker = 9-18 min wall time, acceptable |
| stdlib token bucket (not `pyrate-limiter`) | Solves concurrent-access problems absent in sequential jobs |
| `count_delivered_today()` from DB (not memory) | Safe when multiple jobs run on same day; memory state would be wrong for second job |
| Phase A logs-only for Discovery (no alerts) | Two-week observation window before connecting to router ‚Äî build intuition before going live |
| `headline_dedup_key` promoted to public API | Was private `_headline_dedup_key`; used cross-phase in `news_morning.py` ‚Äî promoted to `classifier/__init__.__all__` |
| `validate_digest_payload()` before email send | Fail-close: if signal counts diverge from digest counts, raise before email rather than send incorrect digest |
| Windows Task Scheduler (not GitHub Actions/cron) | Runner is a Windows machine; no containerization required |
